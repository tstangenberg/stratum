# Security Policy

## Supported Versions

Only the latest release and the `main` branch receive security fixes.
Older releases are not patched — please upgrade to the latest version.

## Reporting a Vulnerability

**Do not open a public issue for security vulnerabilities.**

Use one of the following private channels:

1. **GitHub private vulnerability reporting** — click the "Report a vulnerability" button on the [Security](https://github.com/tstangenberg/stratum/security) tab.
2. **Email** — send details to [thorben@stangenberg.net](mailto:thorben@stangenberg.net).

### What to include

- Description of the vulnerability
- Steps to reproduce
- Affected versions or commits
- Impact assessment (if known)

### Response timeline

| Step | Timeframe |
|---|---|
| Acknowledgement | Within **48 hours** |
| Fix timeline communicated | Within **7 days** |

We will work with you to understand and address the issue before any public disclosure.
