# US-0029: oapi-codegen generation and drift detection in CI

**As a** contributor  
**I want** CI to verify that the generated Go code matches the current OpenAPI spec  
**So that** the spec and implementation stay in sync and drift is caught immediately

## Context

`oapi-codegen` generates `internal/api/api.gen.go` from `api/openapi.yaml` (see ADR-1013). The generated file is committed to the repository. If a contributor updates the spec but forgets to regenerate, or edits the generated file directly, the implementation silently diverges from the contract.

CI detects drift by regenerating the file and comparing it to the committed version. Any diff means the spec and generated code are out of sync.

## Acceptance Criteria

- [ ] CI installs `oapi-codegen` at the pinned version specified in `tools.go` (or `go.mod` toolchain)
- [ ] CI runs `oapi-codegen --config api/oapi-codegen.yaml api/openapi.yaml` and compares the output to `internal/api/api.gen.go`
- [ ] The build fails if there is any diff between the freshly generated file and the committed file
- [ ] The failure message clearly states: "Run `oapi-codegen --config api/oapi-codegen.yaml api/openapi.yaml` to regenerate"
- [ ] This step runs after spec validation (US-0028) — a lint failure stops the pipeline before generation
- [ ] `internal/api/api.gen.go` includes a `// Code generated by oapi-codegen` header so it is clearly marked as generated

## E2E Tests

None — verified by updating `api/openapi.yaml` without regenerating and confirming CI fails with a clear diff message.
