# US-0005: Create a record

**As a** developer  
**I want** to create records via GraphQL mutations  
**So that** I can persist domain data through the API without writing SQL

## Context

The `create` mutation is generated automatically for every type in the uploaded schema. Stratum generates a unique ID for each record. The MVP-1 example creates a `Kanton` record, exercising the `String` and `ID` scalar plugins. See ADR-1007 (no built-in types) and ADR-1008 (plugin architecture).

## Acceptance Criteria

- [ ] `mutation { <type> { create(input: {...}) { id ... } } }` persists the record and returns it
- [ ] `id` is auto-generated by Stratum if not provided in the input
- [ ] All MVP scalar types are accepted: `String`, `Int`, `Float`, `Boolean`, `ID`
- [ ] Required fields (`!`) missing from input return a GraphQL error
- [ ] Created record is immediately queryable
