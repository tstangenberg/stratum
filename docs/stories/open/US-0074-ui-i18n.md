# US-0074: UI — Mehrsprachigkeit (i18n)

**As a** Stratum-Nutzer  
**I want** die Web-Oberfläche in meiner Sprache nutzen können  
**So that** ich Stratum auch ohne Deutschkenntnisse bedienen kann

## Context

Aktuell sind alle user-facing Texte in der UI (HTML-Templates, JavaScript-Fehlermeldungen) fest auf Deutsch verdrahtet. Go-Quellcode und Fehlermeldungen sind auf Englisch. Solange nur eine Sprache existiert, ist das kein Problem — sobald eine zweite Sprache hinzukommt, braucht es eine strukturierte Lösung.

Diese Story führt einen minimalen i18n-Mechanismus ein: Texte werden aus sprachspezifischen Dateien geladen, die aktive Sprache wird über einen konfigurierbaren Default gesteuert. Für den MVP sind Deutsch und Englisch ausreichend.

## Acceptance Criteria

- [ ] Alle user-facing Texte in HTML-Templates und `console.js` sind externalisiert (keine hardcodierten Strings mehr in Templates oder JS)
- [ ] Deutsch und Englisch sind als Sprachen enthalten
- [ ] Die aktive Sprache ist über eine Konfigurationsvariable steuerbar (z. B. `STRATUM_UI_LANG`)
- [ ] Unbekannte Sprachcodes fallen auf Englisch zurück
- [ ] Neue Texte können durch Hinzufügen einer Übersetzungsdatei ergänzt werden, ohne Go-Code zu ändern
